# BiliFocus 人工验收清单

本清单用于快速确认当前版本是否还能正常完成核心工作流。

## 1. 验收准备

1. 运行：

```bash
./infra/checks/prepare-human-validation.sh
```

2. 打开：

- `http://127.0.0.1:3000`
- `http://127.0.0.1:8000/health`

3. 确认健康检查返回 `ok`

## 2. 内容探索页

1. 打开首页
2. 确认页面头部、页签、搜索区域能正常显示
3. 输入关键词，例如：`fastapi`
4. 输入筛选条件，例如：`只看教程，排除直播切片`
5. 点击搜索

需要确认：

- 搜索时有加载状态
- 返回后能看到结果卡片
- 查询上下文区域会显示结果数
- 没有明显报错或空白块

## 3. 同步到本地片库

1. 在搜索结果里执行同步

需要确认：

- 页面出现同步成功提示
- 提示中包含新增数或跳过数
- 同步后进入片库能看到内容

## 4. 智能获取

1. 切到“智能获取”页签
2. 输入一个学习目标
3. 输入排除条件
4. 点击开始智能获取

需要确认：

- 顶部按钮进入执行中状态
- 编排流会按后端阶段推进
- 阶段顺序与后端一致：`planner -> collector -> reviewer -> classifier -> sync`
- 完成后可看到推荐关键词、通过数和结果列表

## 5. 本地片库

1. 打开 `/library`
2. 确认至少有一条本地内容
3. 修改排序为 `发布时间`
4. 输入一个关键词进行本地筛选
5. 点击某条内容进入详情页

需要确认：

- 片库列表能正常渲染
- 排序和筛选会生效
- 分类导航可点击
- 翻页不会报错

## 6. 视频详情

1. 在详情页检查标题、作者、摘要、标签、基础元数据
2. 刷新页面
3. 如果存在播放信息，检查播放器区域是否还能正常显示

需要确认：

- 刷新后页面不崩
- 来源链接仍指向 `bilibili.com/video/...`
- 详情区结构完整，没有明显字段丢失

## 7. 偏好设置

1. 打开 `/settings`
2. 修改默认搜索条数、默认筛选文本、片库排序
3. 保存设置

需要确认：

- 保存成功提示出现
- 刷新后设置仍然存在

## 8. Cookie 与二维码登录

1. 在设置页查看 Cookie 区域
2. 手动填写 Cookie 或生成二维码
3. 如果使用二维码登录，确认扫码成功后 Cookie 会回填

需要确认：

- 状态区会显示是否已配置
- 二维码可生成
- 扫码成功后当前设置节点能回填最新 Cookie

## 9. 结构化重写

1. 在设置页触发“结构化重写”

需要确认：

- 按钮进入执行中状态
- 执行完成后显示重写数和跳过数
- 回到片库或详情页能看到更新后的摘要或标签变化

## 10. 验收通过标准

以下条件同时满足，可视为当前版本基本可用：

- 搜索可用
- 同步可用
- 智能获取可用
- 本地片库可浏览
- 详情页可打开
- 设置可保存
- Cookie 和二维码登录可用
- 没有阻断式报错
